# Real-Time Phishing URL Detection Chrome Extension

## Introduction

A Chrome Extension that detects phishing URLs in real time using
machine learning, helping users spot suspicious links before they get
tricked. Built to be lightweight, fast, and understandable for
non-technical users without compromising security or privacy.

---

## Problem

Phishing attacks cause millions of dollars in damages every year. Most
detection systems rely on blacklists, which can't catch new threats
quickly enough, and many ML-based solutions are difficult to understand
or deploy.

**Can a browser extension use machine learning to detect phishing URLs
in real time, without invading anyone's privacy?**

---

## Solution

A Chrome Extension that:

- Analyzes the current tab's URL when the user clicks a button
- Sends it to a local Flask backend, where a trained ML model predicts
  its phishing risk
- Shows a color-coded risk level: Green (Low), Yellow (Medium), Red (High)
- Uses **no personal data**, stores **nothing**, and runs **locally**

---

## Dataset

- [Phishing URL Dataset](https://data.mendeley.com/datasets/vfszbj9b36/1) — Mendeley Data
- Over 450,000 URLs total: 345,738 legitimate, 104,438 phishing (~23%)
- Class imbalance handled by threshold tuning rather than oversampling
- Download the CSV from the link above and save it as `URL dataset.csv`
  in the project root before training

## Model

- Random Forest Classifier, 100 estimators
- Custom decision threshold of 0.3 (instead of the default 0.5) to
  catch more phishing URLs at the cost of some false positives
- Accuracy: 92%
- Precision (phishing): 89%
- Recall (phishing): 76%
- F1 Score: 0.82

## Features Used

14 handcrafted features extracted from each URL, including:

- IP address used instead of a domain name
- Use of URL shortening services (bit.ly, etc.)
- Suspicious symbols (`@`, extra `//`)
- Suspicious keywords (login, verify, secure, account, etc.)
- Subdomain depth
- Digit and special character counts
- File extension checks (e.g. `.exe`)

## Project Structure
```
phishing-checker-extension/
    manifest.json          — Extension permissions and setup
    popup.html               — Extension popup UI
    popup.js                   — Captures the current tab's URL, calls the backend
feature_extraction.py            — URL feature extraction (shared by training and serving)
train_model.py                     — Trains the classifier and saves it to disk
app.py                               — Flask API serving the trained model
test_api.py                           — Standalone script for testing the API directly
phishing_model.pkl                      — Saved trained model (generated by train_model.py)
```

`URL dataset.csv` is not included in this repository due to its size —
download it from the Mendeley link above.

## How to Run

1. Download the dataset from
   [Mendeley](https://data.mendeley.com/datasets/vfszbj9b36/1) and save
   it as `URL dataset.csv` in the project root.

2. Install dependencies:
```
pip install flask flask-cors pandas scikit-learn joblib
```

3. Train the model (run once):
```
python train_model.py
```
This trains the classifier and saves it to `phishing_model.pkl`.

4. Start the backend:
```
python app.py
```

5. Test the API directly (optional):
```
python test_api.py
```

6. Load the extension in Chrome:
   - Go to `chrome://extensions`
   - Enable Developer mode
   - Click "Load unpacked"
   - Select the `phishing-checker-extension` folder

7. Click the extension icon on any page and click "Check URL" to see
   its phishing risk score.

## Backend (Flask API)

- Exposes a `/check` POST endpoint
- Loads the pretrained model from disk on startup (instant startup,
  no retraining needed)
- Extracts features from the submitted URL in real time
- Returns a prediction and risk score to the extension

## Security and Privacy

- No user data stored or tracked
- Minimal permissions (current tab only)
- Secure API communication with CORS enabled
- No external database or cookie access
- Compliant with Chrome Web Store policies

## Testing

- Verified the trained model reproduces the reported accuracy, precision,
  recall, and F1 score on a fresh training run
- Tested the backend API directly with both legitimate and
  phishing-pattern URLs
- Tested the full integration: URL click → backend → Chrome popup UI
- Verified graceful error handling when the backend is unreachable,
  including confirming no JavaScript errors appear in the browser console

## Model Insights

Most important features:
- Subdomain depth
- Number of special characters
- Digits in the URL
- Suspicious keywords

A lower decision threshold (0.3) was chosen deliberately to prioritize
catching more phishing attempts, since missing a real threat is worse
than an occasional false alarm.
